# GO BUS! 기능정의서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | GO BUS! |
| **목적** | 버스 시간표 이미지에서 출발시간을 추출하고, 출발 N분 전에 알람을 받을 수 있는 스마트 출발 알람 서비스 |
| **대상 사용자** | 버스 이용자 (스마트폰 사용자) |
| **플랫폼** | 모바일 웹 (PWA 권장) |

---

## 2. 주요 기능 요약

1. **시간표 이미지 입력**: 카메라 촬영 또는 갤러리/사진첩에서 시간표 사진 업로드
2. **출발시간 추출**: 이미지에서 OCR로 출발시간(시:분) 자동 추출
3. **알람 대상 선택**: 추출된 시간 중 사용자가 알람을 받을 출발시간 선택
4. **알람 목록 표시**: 선택·등록된 출발시간(알람) 목록을 화면에 표시
5. **스마트 알람**: 출발시간 N분 전(기본 5분)에 스마트폰 알람 발생

---

## 3. 기능 상세 정의

### 3.1 시간표 이미지 입력 (F-01)

| 구분 | 내용 |
|------|------|
| **기능 ID** | F-01 |
| **기능명** | 시간표 이미지 입력 |
| **설명** | 사용자가 버스 시간표를 카메라로 촬영하거나, 기존에 저장된 시간표 사진을 선택하여 입력한다. |

**세부 요구사항**

- **F-01-01** 카메라 촬영: 버튼/액션으로 카메라를 띄우고, 사용자가 시간표가 보이도록 촬영 후 확인 시 해당 이미지를 입력으로 사용한다.
- **F-01-02** 갤러리/사진첩 업로드: 파일 선택(또는 이미지 선택)을 통해 사진첩/갤러리에서 시간표 이미지를 선택할 수 있다.
- **F-01-03** 지원 형식: JPEG, PNG 등 일반적인 이미지 형식을 지원한다.
- **F-01-04** 권한: 카메라 사용 시 사용자에게 카메라 권한 요청을 하고, 거부 시 안내 메시지를 표시한다.

**완료 조건**

- 촬영 또는 선택한 이미지가 다음 단계(출발시간 추출)에 전달된다.

---

### 3.2 출발시간 추출 (F-02)

| 구분 | 내용 |
|------|------|
| **기능 ID** | F-02 |
| **기능명** | 시간표 이미지에서 출발시간 추출 |
| **설명** | 입력된 시간표 이미지를 분석하여, 텍스트 중에서 출발시간(시:분 형식)을 추출한다. |

**세부 요구사항**

- **F-02-01** OCR 사용: 이미지 내 텍스트를 인식하기 위해 OCR(광학 문자 인식)을 사용한다.
- **F-02-02** 시간 형식 인식: `HH:MM`, `H시 M분`, `HH.MM` 등 일반적인 시간 표기 패턴을 인식한다.
- **F-02-03** 과거 시간 제외: 현재 시각 이후의 출발시간만 추출 대상으로 한다(당일 기준).
- **F-02-04** 중복 제거: 동일한 시각은 한 번만 표시한다.
- **F-02-05** 정렬: 추출된 시간 목록은 시간순으로 정렬하여 표시한다.
- **F-02-06** 오류 처리: 추출 실패 또는 추출 결과 없음 시 사용자에게 안내 메시지를 표시한다.

**완료 조건**

- 추출된 출발시간 목록이 화면에 표시되고, 사용자가 다음 단계(선택)로 진행할 수 있다.

---

### 3.3 알람 받을 출발시간 선택 (F-03)

| 구분 | 내용 |
|------|------|
| **기능 ID** | F-03 |
| **기능명** | 알람을 받을 출발시간 선택 |
| **설명** | 추출된 출발시간 목록 중에서 사용자가 알람을 받고 싶은 시간을 선택한다. |

**세부 요구사항**

- **F-03-01** 다중 선택: 여러 출발시간을 동시에 선택할 수 있다.
- **F-03-02** 선택/해제: 이미 선택된 시간을 다시 탭하면 선택 해제된다.
- **F-03-03** 알람 등록: “알람 등록” 등 버튼을 누르면 선택된 시간들이 알람 목록에 반영된다.
- **F-03-04** 알람 N분 전 설정: 출발 몇 분 전에 알람을 울릴지 설정할 수 있다(기본값: 5분).

**완료 조건**

- 선택한 출발시간들이 알람으로 등록되고, 알람 목록 화면에 반영된다.

---

### 3.4 알람 목록 표시 (F-04)

| 구분 | 내용 |
|------|------|
| **기능 ID** | F-04 |
| **기능명** | 등록된 출발시간(알람) 목록 표시 |
| **설명** | 사용자가 등록한 알람(출발시간 및 알람 울리는 시각) 목록을 화면에 표시한다. |

**세부 요구사항**

- **F-04-01** 목록 항목: 각 알람에 대해 (1) 출발시간, (2) 알람 시각(출발 N분 전 시각)을 표시한다.
- **F-04-02** 정렬: 알람 시각 순으로 정렬하여 표시한다.
- **F-04-03** 삭제: 개별 알람을 삭제할 수 있다.
- **F-04-04** 당일 기준: 알람은 당일 기준으로 동작하며, 필요 시 “당일 알람”임을 안내할 수 있다.
- **F-04-05** 데이터 유지: 새로고침 또는 재진입 시에도 등록된 알람 목록이 유지되도록 한다(로컬 저장).

**완료 조건**

- 등록된 알람이 목록으로 보이고, 삭제 등 관리가 가능하다.

---

### 3.5 출발 N분 전 알람 발생 (F-05)

| 구분 | 내용 |
|------|------|
| **기능 ID** | F-05 |
| **기능명** | 출발 N분 전 스마트폰 알람 발생 |
| **설명** | 등록된 각 출발시간에 대해, 설정한 N분 전 시각에 알람(소리·진동·알림)이 발생한다. |

**세부 요구사항**

- **F-05-01** 알람 시각: “출발시간 − N분” 시각에 알람이 울린다(기본 N=5).
- **F-05-02** 알람 방식: 소리 재생, 진동(지원 시), 브라우저/시스템 알림(권한 허용 시)을 사용할 수 있다.
- **F-05-03** 알람 해제: 사용자가 알람을 해제할 수 있는 UI를 제공한다.
- **F-05-04** 백그라운드/탭 비활성: 앱이 백그라운드에 있거나 탭이 비활성일 때도 알림이 동작하도록 권한 요청 및 활용을 고려한다(브라우저 정책 내에서).
- **F-05-05** 한 번 울린 알람: 해당 출발시간에 대한 알람이 울린 후에는 자동으로 비활성 처리한다.

**완료 조건**

- 설정한 시각에 알람이 발생하고, 사용자가 인지하고 해제할 수 있다.

## 4. 비기능 요구사항

| 구분 | 내용 |
|------|------|
| **사용성** | 모바일 화면에 최적화된 UI, 큰 터치 영역 |
| **성능** | OCR 처리 시 로딩/진행 상태 표시, 10초 이내 결과 제공 목표 |
| **호환성** | 최신 모바일 브라우저(Chrome, Safari 등) 지원 |
| **데이터** | 알람 데이터는 로컬 저장(예: localStorage), 서버 필수 아님 |
| **오프라인** | 이미지 입력·OCR·알람 등록까지 오프라인 동작 가능하도록 설계 |
